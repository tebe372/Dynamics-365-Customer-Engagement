# Difference between the Basic and Enhanced Collaboration Experience with Teams in model-driven app


Before you can start using the collaboration experience, it needs to be turned on in Common Data Service. For more information, see [Install and set up the Dynamics 365 app for Teams ](teams-install-app.md).

 
 |  |  |
|---------|---------|
|**Basic Collaboration Experience**     |  <ul><li>You need system admin permissions to enable this feature.</li><br/>  <li>When you choose the **Collaborate** option in a model-driven app record page, the connection set-up process between a model-driven app record and Microsoft Teams collaboration channel happens in the Microsoft Teams app.</li><br/> <li>You can manually pin a model-driven app record or view from the Microsoft Teams app. <br/> <br/>**Note:** The option to add recommended members during the manual set-up process from the Microsoft Teams app is not supported.</li></ul>       |
|**Enhanced Collaboration Experience**    | <ul><li>You need Office 365 tenant admin permissions to enable this feature. </li><br/>  <li>When you choose the **Collaborate** option on a model-driven app record or view page, a set-up wizard in Common Data Service will guide you through the connection set-up process between a model-driven app record or view page and Microsoft Teams collaboration.</li><br/> <li>You can manually pin a model-driven app record or view from the Microsoft Teams app and have the option to add recommended members if you are the team owner.</li><br/>  </ul>
